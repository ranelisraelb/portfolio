Demonstrating the basics.

One tip I always hear from developers is to make sure you have solid HTML & CSS skills. Often times, HTML & CSS are overlooked as being easy so people don't practice writing and end up writing sloppy, non-responsive code that bugs out in between device widths.

While my portfolio might come off as being simple, I was more focused on writing semantic, W3C validated code that would be conceptually modular and easily manageable when I add to it as time goes on.

For now, I'm happy with where it's at and I believe it gives potential employers confidence that my HTML & CSS skills are solid. For this website, I used flex-box for responsiveness and Photoshop to resize images to scale properly in different sized divs.
